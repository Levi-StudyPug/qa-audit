## SECTION 2: CTA COMPLIANCE AUDIT (CRITICAL PRIORITY)

### 2.1 Banned CTA Phrases (Flag as CRITICAL)

**NEVER ALLOWED anywhere on page:**
- "Try Free"
- "Start Free Trial"
- "Start Now For Free"
- "Try for Free"
- "Get Free Access"
- "Free Trial"
- "Try Today" (borderline - avoid)

**Rationale:** StudyPug has NO free trial. It's freemium with hidden daily limits (10 problems, 3 videos, 3 AI searches). These phrases create "bait and switch" perception causing 37%+ signup abandonment.

### 2.2 Header CTA Standard (CRITICAL)

**Rule:** Header CTA must be login-state aware, NOT audience-specific.

| User State | Required Header CTA | Target |
|------------|---------------------|--------|
| Logged out (ALL pages) | "Get Started" | /signup |
| Freemium user | "See Plans" | /payment |
| Paid user | Avatar only | /dashboard |

**Flag as CRITICAL if:**
- Header shows "Try Free", "Start Learning", or any other CTA when logged out
- Header CTA varies by audience (should be unified "Get Started")

### 2.3 Body CTA Standards (By Audience)

Body CTAs CAN be audience-specific:

| Context | Required Body CTA |
|---------|-------------------|
| Parent K-9 pages | "Get Started" |
| Student 10-12 pages | "Start Learning" |
| Homeschool pages | "Start Today" |
| Test Prep pages | "Start Now" |
| University pages | "Start Now" |
| Regional catch-all | "Get Started Today" |
| Course pages | "Start This Course" |
| Mid-page secondary | "See What's Included" or "Browse Topics" |

### 2.4 CTA Context Rules

**Scroll CTA vs Navigation CTA:**
- Hero CTA that scrolls to content: Can use "See What's Included"
- CTA that navigates to signup: Must be "Get Started" or "Start Learning"
- Never use same text for different actions on same page

### 2.5 Meta Description Standard

| Ending | Status |
|--------|--------|
| "Start today." | ✅ APPROVED |
| "Get started today." | ✅ APPROVED |
| "Try today." | ⚠️ AVOID |
| "Try free." | ❌ NEVER |

## SECTION 4: COPY QUALITY AUDIT (HIGH PRIORITY)

### 4.1 Cognitive Load Standards

| Element | Max Length | Requirement |
|---------|------------|-------------|
| How StudyPug descriptions | 45-60 chars | ONE idea per description |
| Why StudyPug descriptions | 50-65 chars | ONE idea per description |
| Regional Why descriptions | 45-55 chars | ONE idea per description |
| Testimonial body | 95-135 chars | Specific outcome + timeline |
| H1 headline | 25-40 chars | Outcome or problem focused |
| H2 subheadline | 50-80 chars | Feature-forward, mechanism |

**Flag as HIGH if:**
- Descriptions exceed character limits
- Descriptions contain multiple ideas (watch for "and", comma lists)
- Descriptions contain topic lists like "fractions, decimals, algebra"

**Topic List Rule:** Topic lists belong ONLY in Curriculum Coverage section.

### 4.2 Grammar & Readability

Check for:
- Spelling errors (including regional spelling - see Section 6)
- Grammatical errors
- Awkward phrasing or run-on sentences
- Inconsistent tense (should be present tense)
- Passive voice (prefer active)
- Reading level appropriate to audience (parent vs student)

**Readability targets:**
- Parent copy: Grade 8-10 reading level
- Student copy: Grade 6-8 reading level
- Headlines: Grade 4-6 reading level

### 4.3 Tone & Voice

**StudyPug voice characteristics:**
- Warm but professional
- Confident without arrogance
- Empathetic to pain points
- Action-oriented
- Never condescending

**Flag as MEDIUM if:**
- Copy sounds robotic or generic
- Copy is overly salesy or pushy
- Copy uses jargon parents/students won't understand
- Tone mismatch (student language on parent page or vice versa)

### 4.4 AI Claim Standards

**ALLOWED AI references:**
- "Photo Search" feature (Grade 4+ only)
- "Snap a photo, get the exact lesson"
- "AI finds the right lesson for your homework"

**BANNED AI references:**
- "AI creates custom study plan"
- "AI builds your learning path"
- "AI-powered learning"
- "AI-powered platform"
- "AI" in any non-photo-feature context

**Rationale:** IXL and Khan Academy don't prominently use "AI" in customer-facing copy. Overclaiming AI reduces trust.

**Flag as HIGH if:** AI mentioned outside of photo recognition feature.

## SECTION 7: TESTIMONIAL AUDIT (MEDIUM PRIORITY)

### 7.1 Banned Phrases

**Never use these words in testimonials (even in "before" context):**
- "tears" (even "from tears to triumph")
- "arguing"
- "fighting"
- "nightmare"
- "dreading"
- "screaming"
- "battles"
- "struggle" (overused - limit to 1 per page max)

**Rationale:** These words prime negative emotions and create anxiety associations with the brand.

### 7.2 Required Elements

| Element | Requirement |
|---------|-------------|
| Specific outcome | Grade improvement % or letter grade change |
| Timeline | Realistic (4-8 weeks typical, max 3 months) |
| Role/context | Grade level, location (regional format), parent/student |
| Character limit | 95-135 characters for body text |

**Good example:**
"Went from C- to B+ in just 6 weeks. The video explanations finally made algebra click." — Sarah M., Grade 8, Toronto, ON

### 7.3 Audience Match

| Page Type | Required Testimonials |
|-----------|----------------------|
| K-9 pages | Parent testimonials only |
| Grade 10-12 pages | Student testimonials only |
| K-12 catch-all | Mix of both (at least 1 of each) |
| Homeschool | Parent testimonials only |
| University | Student testimonials only |

## SECTION 8: UI/UX SCREENSHOT AUDIT (IF PROVIDED)

If screenshot(s) are provided, evaluate:

### 8.1 Visual Hierarchy

| Check | Pass Criteria |
|-------|---------------|
| H1 prominence | Largest text, immediately visible |
| CTA visibility | High contrast, stands out from background |
| Above-fold content | H1, H2, primary CTA, trust signal all visible |
| White space | Adequate breathing room, not cramped |
| Reading flow | Natural left-to-right, top-to-bottom progression |

### 8.2 Accessibility

| Check | Standard |
|-------|----------|
| Color contrast | 4.5:1 minimum for body text (WCAG AA) |
| Button contrast | 3:1 minimum for interactive elements |
| Text size | 16px minimum for body text |
| Touch targets | 44px × 44px minimum for mobile buttons |
| Alt text | All images should have descriptive alt text |

### 8.3 Mobile Optimization

| Check | Requirement |
|-------|-------------|
| Responsive layout | No horizontal scroll |
| CTA button | Full-width on mobile, 48px+ height |
| Sticky CTA | Should appear after scrolling past hero |
| Font size | 14px minimum (readable without zoom) |
| Tables | Collapsed or vertical stack (no horizontal scroll) |
| Tap targets | Adequate spacing between clickable elements |

### 8.4 Trust Signals

| Element | Above Fold? | Prominent? |
|---------|-------------|------------|
| Social proof number | Should be yes | Should be yes |
| Star rating or reviews | Recommended | Medium prominence |
| Teacher credential | Recommended | Medium prominence |
| Curriculum alignment | Recommended | Medium prominence |
| Money-back guarantee | Can be below fold | Should be present |

### 8.5 Common UI Issues to Flag

- Broken images or missing assets
- Overlapping text or elements
- Cut-off text or buttons
- Inconsistent button styles
- Too many competing CTAs
- Unclear visual hierarchy
- Low contrast text
- Cluttered sections
- Excessive use of bold/emphasis
- Missing hover/focus states

## SECTION 9: CONVERSION PSYCHOLOGY AUDIT (MEDIUM PRIORITY)

### 9.1 Psychological Flow Check

Each module should address a specific user question in sequence:

| Position | User Question | What Should Answer It |
|----------|---------------|----------------------|
| Hero | "Is this for me?" | Grade-specific H1 + curriculum match |
| First social proof | "Can I trust this?" | Specific testimonial with outcome |
| 3-Box features | "What do I get?" | Core differentiators |
| How it works | "How will this help?" | Clear 3-4 step process |
| Curriculum | "Does it cover what I need?" | Topic cards or coverage stats |
| Price anchor | "Can I afford it?" | vs tutoring comparison |
| Success stories | "Has it worked for others?" | Multiple varied testimonials |
| Final CTA | "Should I do this now?" | Risk removal + urgency |

### 9.2 Price Anchoring

**Should include:**
- Comparison to tutoring ($50-80/hour reference)
- "Less than $X per day" framing
- Family plan value (if applicable)
- Money-back guarantee mention

**Flag as MEDIUM if no price context on page.**

### 9.3 Content Volume

| Metric | Target | Flag If |
|--------|--------|---------|
| Total word count | 800-1,200 words | > 1,500 words |
| Page sections | 6-8 modules | > 10 sections |
| Curriculum display | Collapsed/expandable | Full table visible |
| Mobile scroll | 5-6 screens | > 10 screens |
